OAuth Setup Wizard

🔐

Welcome to OAuth Setup

To send emails through Gmail or Outlook, you need to configure OAuth credentials. This one-time setup enables secure authentication without storing your password.

🔒 Secure & Safe:
  • Credentials stored in your browser's localStorage only
  • No data sent to any server
  • Client-side application with full control

Which email provider do you want to set up?

You can set up one or both providers

Gmail OAuth Setup

1

Create Google Cloud Project

Open Google Cloud Console ↗

Create a new project or select an existing one

2

Enable Gmail API

Navigate to APIs & Services > Library

Search for "Gmail API" and click Enable

3

Configure OAuth Consent Screen

Go to APIs & Services > OAuth consent screen

Select External user type and fill in required fields

4

Create OAuth Client ID

Go to APIs & Services > Credentials

Click Create Credentials > OAuth 2.0 Client ID

Select Web application type

5

Configure Authorized Origins

Add your current page URL to Authorized JavaScript origins:

6

Copy Client ID

After creating, copy the Client ID and paste it below:

Format: XXXXXXXXXX-XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX.apps.googleusercontent.com

Outlook OAuth Setup

1

Register App in Azure Portal

Open Azure Portal ↗

Navigate to Microsoft Entra ID (formerly Azure AD)

2

Create New Registration

Click App registrations > New registration

Enter a name (e.g., "CSV Email Sender")

3

Configure Redirect URI

Under Redirect URI, select Single-page application (SPA)

Add your current page URL as a redirect URI:

4

Configure API Permissions

Go to API permissions > Add permission

Select Microsoft Graph > Delegated permissions

Search for and add Mail.Send permission

Click Grant admin consent for your organization

5

Copy Client ID

From the Overview page, copy the Application (client) ID and paste it below:

Format: XXXXXXXX-XXXX-XXXX-XXXX-XXXXXXXXXXXX

✅

Setup Complete!

Your OAuth credentials have been saved. You can now authenticate with your email provider.
